# AI Journal Backend

A REST API backend for an AI-powered journaling app. Users submit journal entries and receive mood analysis and psychological insights generated by Claude AI. Built with Express.js, Supabase, and the Anthropic SDK.

## Features

- JWT authentication via Supabase
- Single-entry mood and insight analysis
- Weekly journal summary with mood trend detection
- Dependency-injected routes for clean unit testing

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Express.js v5 |
| Database & Auth | Supabase (PostgreSQL + JWT) |
| AI | Anthropic Claude (`claude-sonnet-4-6`) |
| Testing | Vitest + Supertest |

## Getting Started

### Prerequisites

- Node.js 18+
- A [Supabase](https://supabase.com) project
- An [Anthropic](https://console.anthropic.com) API key

### Installation

```bash
git clone <repo-url>
cd ai-journal-backend
npm install
```

### Environment Variables

Create a `.env` file in the project root:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ANTHROPIC_API_KEY=your-anthropic-api-key
PORT=3000
```

> Use the **service role key** (not the anon key) for server-side auth verification.

### Running the Server

```bash
npm start          # Production
npm run dev        # Development with nodemon (if installed)
```

The server starts on `http://localhost:3000` by default.

## API Reference

All endpoints except `/health` require an `Authorization: Bearer <token>` header containing a valid Supabase JWT.

### Health Check

```
GET /health
```

**Response**
```json
{ "status": "ok" }
```

---

### Analyse a Journal Entry

```
POST /api/analyse-entry
Authorization: Bearer <token>
Content-Type: application/json
```

**Request Body**
```json
{
  "content": "Today was overwhelming. I had back-to-back meetings..."
}
```

**Response**
```json
{
  "mood": "anxious",
  "moodScore": 3,
  "insights": "You appear to be experiencing decision fatigue. Taking short breaks between tasks may help restore focus and reduce stress."
}
```

| Field | Type | Description |
|-------|------|-------------|
| `mood` | string | Short mood label (e.g. `hopeful`, `drained`) |
| `moodScore` | integer | 1–10 scale (1 = very negative, 10 = very positive) |
| `insights` | string | 1–2 sentence psychological insight |

**Error Responses**

| Status | Reason |
|--------|--------|
| `400` | Missing `content` field |
| `401` | Missing or invalid token |

---

### Weekly Summary

```
POST /api/weekly-summary
Authorization: Bearer <token>
Content-Type: application/json
```

**Request Body**
```json
{
  "entries": [
    "Monday: Felt productive and focused.",
    "Tuesday: Struggled to stay motivated...",
    "Wednesday: Had a great conversation with a friend."
  ]
}
```

Entries can be plain strings or objects with a `content` property.

**Response**
```json
{
  "summary": "This week showed a mix of productivity and emotional strain. You navigated social connection well despite mid-week motivation dips.",
  "dominantMood": "reflective",
  "insights": [
    "You tend to feel more energized after social interactions.",
    "Motivation dips mid-week may benefit from structured breaks.",
    "Your resilience is evident in how you recovered by Wednesday."
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `summary` | string | 2–3 sentence narrative of the week |
| `dominantMood` | string | Single mood label for the week |
| `insights` | string[] | Array of 2–4 behavioral or emotional insights |

**Error Responses**

| Status | Reason |
|--------|--------|
| `400` | Missing or empty `entries` array |
| `401` | Missing or invalid token |

## Testing

```bash
npm test                  # Run all tests
npm run test:watch        # Watch mode
npm run test:coverage     # Coverage report
```

Tests use dependency injection — Supabase and Anthropic clients are mocked, so no real API calls are made during testing.

## Authentication Flow

1. The client obtains a JWT from Supabase after login
2. Every protected request includes `Authorization: Bearer <token>`
3. The server calls `supabase.auth.getUser(token)` to verify the token
4. If valid, the request proceeds; otherwise a `401` is returned
